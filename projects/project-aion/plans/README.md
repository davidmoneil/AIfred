# Plans

**Purpose**: Implementation plans and PRDs for Project Aion development work.

---

## Structure

| Directory | Contents |
|-----------|----------|
| `current/` | Active implementation plans |
| `current/prd-variants/` | PRD test variants |
| `archive/` | Completed or superseded plans |

## What Belongs Here

- Implementation plans (e.g., `2026-01-22-organization-implementation-plan.md`)
- PRD documents
- Execution guides
- Task breakdowns

## What Does NOT Belong Here

- Design specifications → `designs/`
- Exploratory ideas → `ideas/`
- Session work logs → `progress/`
- Execution results → `reports/`

## Lifecycle

1. **Create**: Plan starts in `current/`
2. **Execute**: Plan guides implementation work
3. **Track**: Update checkboxes as work completes
4. **Archive**: When fully executed, move to `archive/` with completion date

## Checklist Hygiene

Planning docs with checkboxes should be reviewed at:
- Session end
- Milestone completion
- PR completion
