# Plans

**Purpose**: Session-level work plans — generalized templates and active session plans.

**Layer**: Mind (knowledge that guides work)

---

## What Belongs Here

- Generalized work plan templates
- Active session work plans
- Task breakdown structures
- Execution frameworks

## What Does NOT Belong Here

- Project-specific implementation plans → `projects/project-aion/plans/`
- Behavioral patterns → `patterns/`
- Procedures for large tasks → `workflows/`

## Hierarchy Context

```
Standards (MUST follow)
    ↓
Patterns (SHOULD follow)
    ↓
Workflows (Large task procedures)
    ↓
Designs (Architecture philosophy)
    ↓
Plans (Session-level work) ← YOU ARE HERE
    ↓
Lessons (Memory)
```

## Usage

1. Consult relevant patterns and workflows first
2. Create or reference a plan for the session's work
3. Track progress during session
4. Archive or update plan at session end
