## What Belongs Here

- Specific problem descriptions
- Step-by-step solutions
- Root cause analysis
- Prevention notes

## What Does NOT Belong Here

- General learnings → `lessons/`
- Behavioral patterns → `patterns/`

## Format

Each troubleshooting document should include:
1. **Problem**: What went wrong
2. **Symptoms**: How it manifested
3. **Root Cause**: Why it happened
4. **Solution**: How to fix it
5. **Prevention**: How to avoid it
